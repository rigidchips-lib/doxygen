100 // D3DXSPRITE flags:
101 // -----------------
102 // D3DXSPRITE_DONOTSAVESTATE
103 // Specifies device state is not to be saved and restored in Begin/End.
104 // D3DXSPRITE_DONOTMODIFY_RENDERSTATE
105 // Specifies device render state is not to be changed in Begin. The device
106 // is assumed to be in a valid state to draw vertices containing POSITION0,
107 // TEXCOORD0, and COLOR0 data.
108 // D3DXSPRITE_OBJECTSPACE
109 // The WORLD, VIEW, and PROJECTION transforms are NOT modified. The
110 // transforms currently set to the device are used to transform the sprites
111 // when the batch is drawn (at Flush or End). If this is not specified,
112 // WORLD, VIEW, and PROJECTION transforms are modified so that sprites are
113 // drawn in screenspace coordinates.
114 // D3DXSPRITE_BILLBOARD
115 // Rotates each sprite about its center so that it is facing the viewer.
116 // D3DXSPRITE_ALPHABLEND
117 // Enables ALPHABLEND(SRCALPHA, INVSRCALPHA) and ALPHATEST(alpha > 0).
118 // ID3DXFont expects this to be set when drawing text.
119 // D3DXSPRITE_SORT_TEXTURE
120 // Sprites are sorted by texture prior to drawing. This is recommended when
121 // drawing non-overlapping sprites of uniform depth. For example, drawing
122 // screen-aligned text with ID3DXFont.
123 // D3DXSPRITE_SORT_DEPTH_FRONTTOBACK
124 // Sprites are sorted by depth front-to-back prior to drawing. This is
125 // recommended when drawing opaque sprites of varying depths.
126 // D3DXSPRITE_SORT_DEPTH_BACKTOFRONT
127 // Sprites are sorted by depth back-to-front prior to drawing. This is
128 // recommended when drawing transparent sprites of varying depths.
